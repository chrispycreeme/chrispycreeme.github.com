Potentiometer Calibration

The device uses two potentiometers for real-time control adjustment:

Motor Speed Pot (Pin 34)
Adjusts the motor speed from 0% to 100%. Turn clockwise for faster operation, counterclockwise for slower. This controls the feed dispensing rate.
Starting Position Pot (Pin 35)
Adjusts the initial servo position before operation. This allows precise calibration of the gate or feed opening mechanism.

Display Interpretation

The OLED display shows:

  • Connection Status: WiFi connected/disconnected indicator
  • Cycle Count: Number of completed feed cycles
  • Detection Status: Active monitoring indication
  • IP Address: Local network address of the main module

Device Operation

Automatic Feed Cycle

The device operates through automated cycles controlled by AI-powered detection:

  1. Detection Phase

    Camera captures image and sends to Gemini AI for analysis

  2. Decision Phase

    AI determines if target detected (returns DETECTED or NOT_DETECTED)

  3. Activation Phase (if detected)

    Motor engages and runs for the configured duration (default 2 seconds)

  4. Servo Positioning

    Servo 1 moves to drop position (90¬∞), Servo 2 adjusts as configured

  5. Relay Engagement

    Auxiliary relay activates for 5 seconds

  6. Cycle Reset

    System returns to idle state and waits for next detection

Troubleshooting

‚ùå WiFi Connection Failed
  • Verify "projectkj" network is broadcasting on 2.4GHz (not 5GHz)
  • Check WiFi password is exactly "projectkj"
  • Restart both modules (power cycle)
  • Check router distance - move closer if far away
  • Verify no other devices are experiencing connection issues
‚ùå Motor Not Running
  • Check power supply voltage (requires minimum 5V)
  • Verify motor is mechanically free (not stuck)
  • Test with manual button press on Pin 25
  • Check BTS7960 driver enable pins (R_EN_PIN 23, L_EN_PIN 22)
  • Inspect PWM signal on oscilloscope (RPWM Pin 18, LPWM Pin 19)
  • Verify potentiometer isn't set to minimum (0%)
‚ùå Servo Not Moving
  • Verify servo power supply is adequate (requires 5V, 2A+ for dual servos)
  • Check servo signal cables on pins 32 and 33
  • Test servo independently with dedicated servo tester
  • Ensure servo isn't mechanically blocked
  • Check PWM frequency is 50Hz (standard for servos)
  • Verify signal pulse width is in range 500-2400 microseconds
‚ùå OLED Display Not Showing
  • Verify I2C connections (SDA Pin 21, SCL Pin 5)
  • Check I2C address is 0x3C (use I2C scanner if unsure)
  • Ensure I2C pull-up resistors are present (4.7kŒ© typical)
  • Check display power (3.3V)
  • Try reset sequence: Power off 30 seconds, power on
  • Check serial monitor for initialization errors
‚ùå Camera Feed Not Showing
  • Verify camera module is powered separately
  • Check serial output for camera IP address
  • Access camera at http://[CAMERA_IP] in web browser
  • Ensure camera is not blocked or covered
  • Check camera cable connection (OV2640 module)
  • Verify WiFi connection on camera module (check serial logs)
‚ùå Detection Not Triggering Motor
  • Verify camera captures clear images of target
  • Check Gemini API key is valid in camera code
  • Test manual trigger with button press first
  • Check internet connection quality (needed for AI analysis)
  • Verify main module receives detection signal from camera
  • Check serial logs for "DETECTED" messages
‚ùå Relay Not Activating
  • Check relay power connection (requires separate 5V+ source)
  • Verify relay pin 4 is connected to signal input
  • Test relay with manual relay tester
  • Check detection is actually occurring (OLED shows cycle)
  • Inspect relay for mechanical damage
  • Verify relay wiring to controlled device
